### Что физически уже работает

| Слой | Файл / endpoint | Что делает |
|---|---|---|
| Мысль → запись | `bin/say "..."` | Пишет в `replies.jsonl` + `graph_events.jsonl` (event=judgment) + `data/brain/<project>.jsonl` (kind=judgment). Закрывает pending dispatch span с реальным `chars` + `dt_ms`. |
| Атрибуция | `bin/say` v3 | На каждое суждение считает `cities = [project,...]` и `subprojects = {project: [top_dir,...]}` по tool_use в окне (prev_judgment.at .. now]. |
| Транскрипт | `bin/transcript_tail.py` (daemon) | Tail-ит мою Claude Code сессию (`~/.claude/projects/<slug>/<session>.jsonl`), эмитит каждый `thinking` / `text` / `tool_use` в `graph_events.jsonl`. Regex путей теперь работает на ПОЛНОЙ Bash команде. |
| Диспетчер | `bin/langgraph_dispatch.py` | LangGraph StateGraph (router → dispatch → evaluate). Когда выбрана claude — НЕ пишет фейковый "(→ Claude inbox)", а оставляет `node_pending` span. `bin/say` потом закрывает его. |
| API | `server.py` | `/say`, `/log`, `/api/graph`, `/api/brain[/list]`, `/api/hypotheses`, `/api/tree?p=NAME` (новый — рекурсивный список файлов через git ls-files), `/api/meta`, `/p?name=...`. |
| Live status | `bin/claude_tracer.py` (PostToolUse hook) | Пишет каждый tool call (Read/Bash/Edit/...) в `graph_events.jsonl` event=claude_tool. *(дублирует transcript_tail; оба остаются для надёжности).* |
| Главная UI | `index.html` спойлер «🕸 Графы» | Tab-bar над iframe: 🕸 LangGraph · 🧠 Суждения · 📊 Brain · 🌐 Проекты. Клик меняет iframe.src. |
| Графы | `graphs/*.html` | `graph.html` (LangGraph dispatch + история запусков, tap-handler для мобилы), `reasoning.html` (live-фид), `judgments.html` (список), `brain.html` (Cytoscape граф судьб + панель с tool_use следом), `project_files.html` (Cytoscape граф файлов проекта, по `/api/tree`), `projects.html` (hub из 5 карточек). |

### Что я понял из сегодняшнего разговора

- **Город = репо** (1 из 5: depz-toolkit, istereolab-sdk, depz-camera-sdk,
  ifirmware-stereocam, iproject_menger).
- **Под-проект внутри города = top-папка** (например `bindings/`, `src/stereo/`,
  `processing/`). Иерархия 2 уровня — это то что мы пока строим.
- **Суждение = морфизм** (input → transformation → output). В нём *преобразуется*
  информация, которую ты дал — поэтому атрибуция к городу/папке нужна, иначе теряем
  ниточку: что именно эта мысль трансформировала.

## 1. Открытые нюансы (что НЕ так / что недоделано)

### A) Навигация при клике
- Сейчас клик по ноде графа открывает правую панель с метаданными (это ок). Кнопки в
  панели уходят в `_top` (заменяют страницу). Это сбивает контекст: ты выпадаешь из
  графа.
- **Возможно**: открывать содержимое файла inline (под графом или в той же панели),
  либо в новой вкладке (target=_blank). Ты выбрал «объясню текстом» — жду что точно.

### B) Привязка нескольких городов к одной мысли
- Сейчас `about` пишется как ОДИН активный проект, плюс `cities[]` массив. Это ок,
  но `data/brain/<project>.jsonl` всё ещё разнесён по файлам — суждение про несколько
  городов попадёт ТОЛЬКО в файл активного. Надо или дублировать в каждый,
  или сделать единый brain.jsonl с фильтром по cities.

### C) thinking-блоки от API зашифрованы
- 46 thinking-блоков в моей сессии — все `thinking=""`, только `signature`. Это
  политика API. Не получится получить «открытым текстом моё думание». Заменяет это —
  `text` блоки (короткие сообщения в Claude Code) и `tool_use` следы — они идут
  через transcript_tail.

### D) Старый /graph.html
- Имеет историю запусков + вкладки Dispatch/Claude шаги, но tap-handler был только
  что добавлен мной — раньше клик на мобиле ничего не делал. На моих новых страницах
  tap работает изначально.
